# Переход с RC.6 на Rules 4.0.0 / RC.7

Rules 4.0.0 принимает только `jsonspecs/spec` 1.0.0-rc.7. Режима совместимости или
автоматической подмены версии RC.6 нет.

1. Измените `specVersion` снэпшота на `"1.0.0-rc.7"`.
2. Пересоберите весь снэпшот и заново вычислите `sourceHash`.
3. Выполните все 309 проверок совместимости RC.7.

Снэпшот RC.6 отклоняется с кодом `UNSUPPORTED_SPEC_VERSION`, даже если исполняемый
граф артефактов в остальном не изменился.

RC.7 закрывает пробел переносимости точных индексных сегментов. Движок сохраняет их
исходную десятичную запись при построении конкретного пути ошибки и не использует
небезопасное число JavaScript для обращения к массиву. Новых операторов, полей DSL,
полей снэпшота, полей результата и новой `formatVersion` не добавлено.

Полное поведение задано в закреплённых
[спецификации RC.7](https://github.com/jsonspecs/spec/blob/f474b5924b55e20e61a8760f0ea752d630ccdf69/SPEC_RU.md)
и [руководстве по переходу](https://github.com/jsonspecs/spec/blob/f474b5924b55e20e61a8760f0ea752d630ccdf69/MIGRATION_RC7_RU.md).
